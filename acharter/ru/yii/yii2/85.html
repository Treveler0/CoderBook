Функциональные тесты

В простых терминах мы устанавливаем переменные $_REQUEST, $_GET и $_POST, а затем мы выполняем приложение из теста.

Это может быть полезно, поскольку функциональные тесты быстрее и обеспечивают подробные следы стека при сбоях.

Codeception может подключаться к различным фреймворкам PHP, которые поддерживают функциональное тестирование: Symfony2, Laravel5, Yii2, Zend Framework и другие.

Вам просто нужно включить нужный модуль в конфигурации вашего функционального пакета.

Модули для всех этих фреймворков имеют один и тот же интерфейс, поэтому ваши тесты не привязаны ни к одному из них.

Это пример функционального теста:

    <?php
    // LoginCest.php
    
    class LoginCest
    {
        public function tryLogin (FunctionalTester $I)
        {
            $I->amOnPage('/');
            $I->click('Login');
            $I->fillField('Username', 'Miles');
            $I->fillField('Password', 'Davis');
            $I->click('Enter');
            $I->see('Hello, Miles', 'h1');
            // $I->seeEmailIsSent();
        }
    }

Как вы видите, синтаксис одинаковый для функциональных и приемочных тестов.

Тесты Yii2 включены в базовые и расширенные шаблоны приложений.

Следуйте инструкциям Yii2 для запуска.

Сама по себе структура Yii не имеет механизма для функционального тестирования.

Таким образом, Codeception является первой и единственной функциональной платформой тестирования для Yii.

Чтобы использовать его с Yii, включите модуль Yii1 в конфигурацию:

    # functional.suite.yml
    
    actor: FunctionalTester
    modules:
        enabled:
            - Yii1
            - \Helper\Functional

Чтобы избежать общих ошибок, которые мы обсуждали ранее, Codeception предоставляет базовые привязки к движку Yii. Пожалуйста, установите их.
