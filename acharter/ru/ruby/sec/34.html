Безопасность среды

За пределами этого руководства осталось, как обезопасить код приложения и среды (environments).

Однако, пожалуйста, обеспечьте безопасность конфигурации вашей базы данных, т.е. config/database.yml, и секретного ключа сервера, хранящегося в config/secrets.yml.

Для дальнейшего ограничения доступа используйте специфичные для сред версии этих и любых других файлов, которые могут содержать деликатную информацию.

Настраиваемые учетные данные

Rails генерирует config/credentials.yml.enc для хранения сторонних учетных данных внутри репозитория.

Это единственно возможный способ, потому что Rails шифрует файл с помощью мастер-ключа, который сгенерирован в системе контроля версии, проигнорировав config/master.key — Rails будет также искать этот ключ в ENV["RAILS_MASTER_KEY"].

Rails еще требует, чтобы ключ загружался в production, поэтому учетные данные могут быть прочтены.

Для редактирования хранящихся учетных данных используйте bin/rails credentials:edit.

По умолчанию этот файл содержит секретный ключ приложения secret_key_base, но он может использоваться также для хранения других учетных данных, таких как ключи доступа к внешним API.

Учетные данные, добавленные в этот файл, доступны через Rails.application.credentials.

Например, со следующим дешифрованным config/credentials.yml.enc:

    secret_key_base: 3b7cd727ee24e8444053437c36cc66c3
    some_api_key: SOMEKEY

Rails.application.credentials.some_api_key вернет SOMEKEY в любой среде.

Если вы хотите вызвать исключение, когда ключ пустой, используйте версию с восклицательным знаком:

    Rails.application.credentials.some_api_key! # => вызовет KeyError: key not found: :some_api_key
